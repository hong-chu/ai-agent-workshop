The Basics

Foundational concepts you'll see everywhere
Artificial Intelligence (AI)
Basic
The broad field of building machines that can perform tasks that typically require human intelligence — understanding language, recognizing images, making decisions, solving problems.
AI is the umbrella term. Machine learning, deep learning, and LLMs are all subfields within AI — like how "sports" includes basketball, soccer, and tennis.
Parameter aka "weight"
BasicPart 1
A single adjustable number inside a neural network. A model with 175 billion parameters (like GPT-3) has 175 billion knobs that were tuned during training. More parameters generally = more capacity to learn patterns.
If a model is a piano, parameters are the individual strings. More strings = richer sound. GPT-1 had 117 million strings. GPT-4 has roughly 1.8 trillion.
Training
Basic
The process of teaching a model by showing it enormous amounts of data (for LLMs: most of the internet's text). The model adjusts its parameters to get better at predicting the next word. Training GPT-4 reportedly cost $100M+ in compute.
Training is like studying for an exam by reading every book in every library. You don't memorize the books — you learn patterns, grammar, facts, and reasoning from them.
Inference
Basic
Using a trained model to make predictions or generate outputs. When you type a question to ChatGPT and it responds, that's inference. Training happens once (expensively); inference happens millions of times per day.
Training = studying for the exam. Inference = taking the exam. You study once, but you take many exams.
GPU Graphics Processing Unit
BasicPart 2
The hardware that powers AI training and inference. Originally designed for video games, GPUs excel at doing millions of simple math operations in parallel — exactly what neural networks need. NVIDIA's H100/B200 GPUs are the gold standard.
A CPU is like one very fast mathematician. A GPU is like 10,000 average mathematicians working at the same time. AI needs the 10,000.
Open Source vs. Closed Source
Part 2
Open source: the model weights are publicly available — anyone can download, use, and modify the model (e.g., Llama, DeepSeek). Closed source: only accessible via API — the company keeps the weights secret (e.g., GPT, Claude, Gemini).
Open source = a recipe you can cook yourself and tweak. Closed source = a restaurant where you order the dish but can't see the recipe.

Architecture

How models are structured inside
Token
Part 1
The basic unit an LLM processes. Not exactly a word — tokens are pieces of words created by Byte-Pair Encoding (BPE). Common words like "the" are one token. Uncommon words get split: "tokenization" → "token" + "ization". On average, 1 token ≈ 0.75 words.
If words are LEGO sets, tokens are the individual LEGO pieces. Common sets stay whole; uncommon ones get broken into reusable parts.
Embedding
Part 1
Converting a token into a list of numbers (a vector) that captures its meaning. Each token gets looked up in a giant learned table (~50,000 tokens × ~12,000 dimensions). Words with similar meanings end up close together in this number space.
"king"
→
[0.21, -0.87, 0.53, ...]
~12,000 numbers
GPS coordinates for meaning. "King" and "queen" are nearby. "King" and "potato" are far apart. The model learned these positions during training.
Transformer
Part 1
The neural network architecture behind all modern LLMs (2017). Its key innovation: self-attention, which lets every word look at every other word simultaneously. This replaced sequential models (RNNs) and unlocked parallelism, scaling, and long-range understanding.
Older models read like a person scanning left-to-right, one word at a time. A Transformer reads like someone who can see the entire page at once and instantly understand which words relate to which.
Self-Attention aka "Attention mechanism"
Part 1
The core mechanism of Transformers. For each word, it computes a relevance score with every other word using three learned vectors: Query (what am I looking for?), Key (what do I contain?), and Value (what information do I carry?). The formula: Attention = softmax(Q × K^T / √d) × V.

At a party, self-attention is like being able to hear every conversation simultaneously and focusing on the ones most relevant to what you're saying. Q = "I need context about X." K = "I have info about Y." If X and Y match, the attention score is high.
Context Window aka "Context length"
Part 1
The maximum number of tokens a model can process in a single conversation. GPT-4: 128K tokens (~300 pages). Claude: 200K tokens. Gemini: 1M+ tokens. Longer context = can read more material at once, but costs more compute.
The context window is the model's working memory — how many pages of notes it can spread out on its desk at once. A bigger desk = can work with more information simultaneously.

Training & Learning

How models learn and improve
Pre-training
Part 1
The first, most expensive phase of training where the model reads a massive corpus of text (books, websites, code) and learns to predict the next token. This gives the model its general knowledge, language ability, and reasoning foundations. Costs tens to hundreds of millions of dollars.
Pre-training is like going through all of school — kindergarten through PhD. You build broad knowledge about everything, not yet specialized for any particular job.
Fine-tuning
Part 1
Additional training on a smaller, specific dataset to specialize the model for a particular task or behavior. For example, fine-tuning on medical texts to create a medical AI, or fine-tuning on conversation examples to make the model a better chatbot.
If pre-training is going to medical school, fine-tuning is your residency — specializing in cardiology by doing hundreds of cardiac procedures.
RLHF Reinforcement Learning from Human Feedback
Part 1
A technique that makes models helpful and safe by training them on human preferences. Step 1: Humans rank model outputs (response A is better than B). Step 2: A "reward model" learns these preferences. Step 3: The LLM is trained via RL to maximize the reward. This is what turned raw GPT-3 into ChatGPT.

Helpful chatbot
Training a dog: the dog (model) tries something, you say "good boy" or "bad boy" (human feedback), and eventually it learns what you want.
Scaling Laws
Part 1
The 2020 discovery that model performance improves as a predictable power law of three variables: model size (parameters), training data, and compute. This meant researchers could predict how good a model would be before training it — turning AI from trial-and-error into engineering.
If you know that doubling your study hours improves your test score by 5%, you can plan exactly how long to study for the grade you want. Scaling laws gave AI researchers the same kind of predictability.
Few-shot / Zero-shot Learning
Part 1
Zero-shot: the model performs a task it was never specifically trained on, just from the instruction. Few-shot: you give 2-3 examples in the prompt and the model follows the pattern. GPT-3 was the first model to do this well, which was a major surprise.
Zero-shot = "Translate this to French" (no examples). Few-shot = "Here's how I'd translate: 'Hello' → 'Bonjour', 'Goodbye' → 'Au revoir'. Now translate: 'Thank you'" — and it gets it.
Chain-of-Thought (CoT)
Part 1Part 3
A prompting technique (and later, a training technique) where the model shows its reasoning steps before giving a final answer. Adding "Let's think step by step" to a prompt dramatically improves accuracy on math, logic, and reasoning tasks.
Instead of blurting out "42!" the model writes: "First I need to... then I calculate... then I check... the answer is 42." The intermediate steps become context that helps the next prediction be more accurate — like giving yourself scratch paper on an exam.

Models & Families

The major model types and what they do
LLM Large Language Model
Basic
A neural network with billions of parameters trained on massive text data to understand and generate language. "Large" = billions of parameters. All modern AI chatbots (ChatGPT, Claude, Gemini) are LLMs. Under the hood, they work by predicting the next token, one at a time.
A giant pattern-matching engine. It's read so much text that it's very good at predicting what should come next — and that ability to predict turns out to be surprisingly close to "understanding."
GPT Generative Pre-trained Transformer
Part 1Part 2
OpenAI's model family. Reads left-to-right and predicts the next word. "Generative" = it creates new text. The approach that won: GPT-1 (117M) → GPT-2 (1.5B) → GPT-3 (175B) → GPT-4 (~1.8T) → GPT-5. Plus reasoning specialists: o1 → o3.
BERT Bidirectional Encoder Representations from Transformers
Part 1
Google's 2018 model. Reads in all directions (bidirectional) and was trained to fill in masked words. Excellent at understanding and classification tasks (search, sentiment), but cannot generate new text. Lost to GPT because generation proved more versatile than understanding.
Multimodal
Part 1Part 2
A model that can process multiple types of input — text, images, audio, video. GPT-4 was the first major multimodal LLM (text + images). Gemini processes text, images, audio, and video. This is a shift from "language model" to "everything model."
A multilingual person who can also read sheet music, interpret photos, and lip-read. Not just text anymore — the model can see and hear.
Reasoning Model e.g., o1, Claude Extended Thinking
Part 1Part 3
A model trained via RL to think step-by-step internally before answering. Unlike CoT prompting (which just asks the model to show its work), reasoning models have internalized the habit of decomposing problems, checking work, and backtracking. They spend more "thinking tokens" on harder problems.
Regular model = a student who blurts out the first answer that comes to mind. Reasoning model = a student who's been trained to always show their work, double-check, and only then give the answer.

Alignment & Safety

Making AI do what we actually want
Alignment
Part 1Part 2
The challenge of making AI systems behave in ways that match human intentions and values. A misaligned model might be very capable but optimize for the wrong things — like a genie that grants wishes too literally. RLHF, Constitutional AI, and safety testing are all alignment techniques.
You hire a brilliant assistant but need to make sure they actually help you — not just complete tasks technically correctly while doing harm in the process. Alignment is the process of teaching them your actual values.
Constitutional AI
Part 2
Anthropic's approach to alignment. Instead of training on raw human preferences (RLHF), the model is trained against a written set of principles (a "constitution") — like "be helpful, harmless, and honest." The model critiques and revises its own outputs based on these principles.
Instead of a teacher grading every essay, you give the student a rubric and teach them to grade themselves. The constitution is the rubric.
Hallucination
Basic
When a model confidently generates information that is factually wrong or made up. Because LLMs predict the most likely next token (not truth), they can produce fluent, convincing text about things that never happened, papers that don't exist, or statistics they invented.
A very confident friend who always has an answer — even when they're guessing. They sound so sure you don't think to fact-check, but sometimes they're completely making it up.
Guardrails
BasicPart 2
Safety measures built into models to prevent harmful outputs — refusing to provide dangerous information, avoiding bias, declining inappropriate requests. Different companies implement guardrails differently: some stricter (Claude), some more permissive.

Agents & Tools

Making AI do things in the real world
Function Calling aka "Tool use"
Part 1Part 3
The model outputs a structured JSON command instead of regular text, telling a host program to execute an action (search the web, read a file, call an API). The host runs the action and sends the result back. This is how LLMs go from "text generators" to "tool users."

AI Agent
Part 1Part 3
An AI system that can autonomously plan, use tools, and complete multi-step tasks in a loop: Think → Act → Observe → Repeat. Unlike a chatbot (human drives), an agent drives its own workflow. You give it a goal; it figures out the path.

A chatbot is like texting a smart friend — you ask questions, they answer. An agent is like hiring an assistant — you say "book me a flight to Tokyo" and they handle the research, comparison, and booking autonomously.
MCP Model Context Protocol
Part 3
An open standard (by Anthropic) for connecting AI models to external data sources and tools. Think of it as a universal adapter — instead of building custom integrations for every tool, MCP provides a standard way for AI to talk to databases, APIs, file systems, and more.
USB-C for AI. Before USB-C, every device had a different charger. MCP is one standard protocol so any AI can connect to any tool.
Prompt / Prompt Engineering
BasicPart 3
The text you give to an AI model to get it to do what you want. Prompt engineering is the skill of crafting prompts that produce the best results — using specific instructions, examples, formatting, and context to guide the model's output.
The prompt is like a brief you give a freelancer. A vague brief ("write something about dogs") gets mediocre results. A specific brief ("write a 200-word blog post about golden retrievers for first-time dog owners, in a warm tone") gets exactly what you want.
RAG Retrieval-Augmented Generation
Part 3
A technique where the model retrieves relevant documents from a knowledge base before generating its answer. Instead of relying only on what it learned during training, the model gets fresh, specific context. This reduces hallucinations and lets the model answer about your private data.
Instead of answering a test from memory alone, you're allowed to look through your notes first. The model searches a database, finds the most relevant passages, and uses them to craft its answer.
Agentic Loop
Part 3
The core execution cycle of an AI agent: Perceive → Reason → Act → Observe, repeated until the task is complete or a stopping condition is met. Unlike a single LLM call, the agentic loop allows the model to take action, see the result, adjust its plan, and keep going — making it capable of multi-step problem solving.

A chef cooking a new recipe: taste, think about what's missing, add an ingredient, taste again. Each loop refines the result until it's right.
Multi-Agent Systems
Part 3
An architecture where multiple specialized AI agents collaborate on a task. Each agent has a specific role (researcher, coder, reviewer) and they pass messages to each other. A supervisor or orchestrator agent coordinates the workflow. This mirrors how human teams work — divide and conquer.

Instead of one person doing everything, you have a project manager assigning tasks to a research team, an engineering team, and a QA team — each plays to their strengths.
Human-in-the-Loop aka "HITL"
Part 3
A design pattern where an AI agent pauses and asks a human for approval before taking critical or irreversible actions. The agent handles routine steps autonomously but escalates high-stakes decisions (spending money, deleting data, sending emails) to a human. Balances automation speed with human judgment.
An assistant who handles your calendar but calls you before canceling an important meeting. They do the routine work, but check in before doing anything you might regret.
Agent Memory aka "State", "Persistent Context"
Part 3
How an agent retains information across interactions and sessions. Short-term memory is the current conversation context. Long-term memory stores preferences, past decisions, and learned facts in a database so the agent remembers you next time. Without memory, every conversation starts from scratch.
Short-term memory is your whiteboard during a meeting — wiped afterward. Long-term memory is your notebook — you can flip back to it months later.
A2A Agent-to-Agent Protocol
Part 3
An open standard (by Google) enabling AI agents from different vendors to communicate with each other. While MCP connects an agent to tools, A2A connects agents to other agents — letting a travel-booking agent coordinate with a calendar agent, even if they were built by different companies.
MCP is like plugging a device into a USB port. A2A is like two people exchanging business cards and agreeing on how they'll work together.
Skills
Part 3
Reusable instruction sets that teach an agent how to do a specific task well — like creating spreadsheets, writing documents, or generating images. Instead of re-explaining the process every time, you package the best practices into a skill the agent can load on demand. Skills turn a general-purpose agent into a specialist.
A recipe card for an AI chef. The chef (agent) is capable of cooking anything, but the recipe card (skill) gives it the exact steps, tips, and tricks for making a perfect soufflé every time.
System Prompt
Part 3
Hidden instructions given to an AI model before the user's conversation begins. It sets the agent's personality, rules, capabilities, and constraints. The user never sees it, but it shapes every response. This is how companies customize the same base model into different products.
The employee handbook your new hire reads on day one. The customer never sees it, but it determines how the employee behaves, what they're allowed to do, and how they handle tricky situations.
