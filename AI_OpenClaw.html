Three Names, One Lobster

A trademark dispute, a viral social network, and the fastest-growing repo in history
NOV 2025
🦞 Clawdbot
Original name. A play on "Claude" + claw. Built by Peter Steinberger as a personal AI assistant. Quiet launch, small community.
→
JAN 27, 2026

Anthropic's lawyers flagged "Clawdbot" (too close to "Claude"). Renamed to "Moltbot" — lobsters molt to grow. The creator said the name "never grew on him."
→
JAN 30, 2026
🦞 OpenClaw
Final rebrand. Same day: crossed 100K stars. Then Moltbook (a social network for AI agents) went viral, driving another wave of attention. Hit 250K by March.
🦞 WHAT IS MOLTBOOK?
On January 28, entrepreneur Matt Schlicht launched Moltbook — a social network designed exclusively for AI agents. Users connect their OpenClaw bots, which then interact with each other, sharing recommendations and completing tasks collaboratively. The viral buzz from Moltbook drove a massive wave of interest in OpenClaw itself.

UNDER THE HOOD
04

How OpenClaw Works

Follow one message from your phone through the entire system

Imagine you text your OpenClaw agent on Discord: "Do I have any meetings tomorrow? If yes, prep a 1-page summary of each topic." Here's exactly what happens behind the scenes — step by step.

1
YOUR MESSAGE ARRIVES
The Gateway catches it
Your Discord message hits the Gateway — a lightweight server running on your computer. It speaks WhatsApp, Signal, Telegram, and Discord. The Gateway strips away platform-specific formatting and turns your message into a clean text the agent can understand. Think of it like a receptionist who answers every phone line.
2
THE AGENT READS ITSELF INTO EXISTENCE
Loading the workspace files
Before thinking about your message, OpenClaw reads a folder of plain text files from your disk — its "DNA." These files tell the agent who it is, who you are, and what it should remember:
SOUL.md
"Be concise. Never use jargon. Ask before doing anything irreversible." — the system prompt, but you wrote it.
USER.md
"Name: Yeesun. Role: Engineer. Prefers Korean greetings." — how the agent personalizes every reply.
memory/2026-03-06.md
Today's log + yesterday's. "Yeesun had a 1-on-1 with Lisa. Discussed Q2 roadmap." — short-term memory.
IDENTITY.md
The agent's name, emoji, personality. Created during initial "bootstrap ritual" setup.
3
THE PROMPT IS ASSEMBLED
Everything gets stitched into one giant prompt
OpenClaw combines all of this into a single prompt and sends it to an LLM (Claude, GPT, DeepSeek — you choose). The prompt looks roughly like this:
[SOUL.md contents]            ← who you are
[USER.md contents]            ← who I am
[IDENTITY.md contents]        ← agent persona
[TOOLS.md contents]           ← how to use my tools
[memory/2026-03-05.md]         ← yesterday
[memory/2026-03-06.md]         ← today so far
[available skills list]        ← what I can do
"Do I have meetings tomorrow?" ← your message
4
THE LLM THINKS AND ACTS
The Agent Loop: reason, act, observe, repeat
The LLM reads the prompt and decides it needs to check your calendar. It calls the calendar skill (a code module installed from ClawHub). The skill fetches tomorrow's events. The LLM sees 3 meetings, then calls the web-search skill to research each topic, then calls a document skill to draft summaries. Each step is: think → call a skill → read the result → think again. This loop repeats until the task is done.

5
YOU GET A REPLY
Back to Discord with the answer
The final answer flows back through the Gateway to your Discord chat. You see: "You have 3 meetings tomorrow. Here are the summaries..." with a document attached. The whole exchange took ~15 seconds. To you, it felt like texting a really fast colleague.
6
THE AGENT REMEMBERS
Today's log gets updated
OpenClaw appends a note to memory/2026-03-06.md: "Yeesun asked about tomorrow's meetings. Prepared 3 summaries." Next time you chat, the agent already knows this happened — no need to repeat yourself.
💓
The Heartbeat: Proactive Mode
Not just reactive — OpenClaw can act on its own
The 6 steps above happen when you message the agent. But OpenClaw can also wake itself up. Every 30 minutes, it reads a special file called HEARTBEAT.md — a checklist of things to watch for. If a condition is met, the agent messages you first.
# Heartbeat — check every 30 min
- If email from my manager → summarize and notify me
- If meeting starts in <30 min → send me a reminder
- If any GitHub PR needs my review → ping me once
- Only message me if something actually needs attention
📁 THE KEY INSIGHT
OpenClaw is only as smart as the prompt it constructs — and you control every file that goes into that prompt. Edit SOUL.md to change who the agent is. Edit HEARTBEAT.md to change what it watches for. Edit memory files to change what it remembers. Every behavior is traceable to a plain text file on your disk. This "workspace-first" design means your agent is portable, version-controllable, and completely transparent.
🏠 LOCAL-FIRST
Everything runs on your computer. The gateway, the skills, and the memory all stay local. Only the LLM reasoning calls go to the cloud (unless you run a local model). This is a key part of OpenClaw's appeal — and its risk: your computer becomes the execution environment, with all its data accessible to the agent.

THE CONTROVERSY
06

The Security Debate

Why Microsoft, Cisco, and Palo Alto Networks published warnings about OpenClaw
🎯
Prompt Injection
If OpenClaw reads a malicious email or web page containing hidden instructions, it could be tricked into executing commands on your system — exfiltrating data, deleting files, or sending unauthorized messages.
⚠️ Researchers at Giskard demonstrated account takeover via a crafted email attachment.
🔓
Excessive Permissions
OpenClaw often runs with full access to your OS — shell commands, file system, email, browser. A single mistake or vulnerability can cascade into total system compromise. 30,000+ exposed instances were found in a 2-week scan.
⚠️ CVE-2026-25253 (CVSS 8.8): Remote code execution chain, patched Jan 30, 2026.
🧩
Malicious Skills (Supply Chain)
Anyone can publish skills to ClawHub. Unvetted or malicious skills may contain code that delivers malware. If OpenClaw has full OS access, a malicious skill can do anything on your machine.
⚠️ No mandatory code review for published skills. Trust model relies on community flagging.
🔑
Plaintext Secrets
API keys, messaging platform tokens, and credentials are stored in plaintext configuration files. If an attacker accesses the file system (via a skill exploit or shared session), they get everything.
⚠️ Shared "main" session design means multiple users can accidentally see each other's data.
⚖️ THE CORE TENSION
OpenClaw is powerful because it has deep system access. But that same access makes it dangerous. This is the fundamental tradeoff of AI agents: the more an agent can do, the more damage it can do if something goes wrong. OpenClaw is the first project to make this tradeoff visible at massive scale — and the security community's response is shaping how all future agents will be built.
THE BIGGER PICTURE
07

Why OpenClaw Matters

Beyond the hype — what it signals about the future of AI agents
📱
Messaging as Interface
OpenClaw proved that people don't want another app — they want AI in the apps they already use. WhatsApp, Signal, Telegram become the control center. This "meet users where they are" approach is now being copied by every major player.
🏠
Local-First AI
Your data stays on your machine. Your agent runs on your hardware. In a world of cloud-first everything, OpenClaw represents the countermovement: AI that's truly personal and private (if configured correctly).
🌍
Open Source Wins (Again)
250K stars in 4 months. Anyone can inspect, modify, and extend the code. The skill marketplace is community-driven. This is the Linux moment for AI agents — proving that open ecosystems move faster than closed ones.
⚠️
The Security Wake-Up Call
Microsoft, Cisco, Palo Alto Networks, and Mastercard all published security advisories. OpenClaw forced the industry to confront: what happens when millions of people give AI agents full access to their computers? The answer is shaping new security standards.
🔧
Skills as the New Apps
ClawHub is an app store for agent capabilities. Install a "travel" skill and your agent can book flights. Install a "coding" skill and it writes code. This plugin model — skills that extend agent capabilities — is becoming the standard pattern across all agent platforms.
🚀
One Person, One Agent
OpenClaw's persistent memory means it learns you over time — your preferences, schedule, communication style. This is the vision: not a generic chatbot, but a personal AI that knows you as well as a long-time assistant would.
🔮 LOOKING AHEAD
In February 2026, creator Peter Steinberger joined OpenAI. The OpenClaw project continues as a community-driven open-source effort. Whether OpenClaw itself becomes the dominant personal agent or simply proved the model, its impact is permanent: the era of personal AI agents has arrived, and there's no going back.
